getData RSO

getData Remaster is not a distributed end-user application. It is a private local administrator tool used by the getData operator to manage manually entered League of Legends custom game records for a small friend group.

Participants may voluntarily authorize Riot Sign On so the operator can read Riot account and custom match data through approved Riot APIs. The operator then imports or reviews that data locally in getData Remaster. Generated summaries are shared through a private Telegram bot to authorized bot users and private group chats.

View Demo Flow

Actual Operating Model

01User opt-in through RSO

Participants explicitly grant access through Riot Sign On before their Riot data is used.

02Local admin processing

The getData operator imports, reviews, and stores match data in a private local getData Remaster database.

03Private Telegram output

Statistics and summaries are shown only to authorized Telegram bot users and private group chats.

What getData Is Not

  • It is not a public website or public leaderboard.
  • It is not a distributed consumer application.
  • It is not commercial and is not used for gambling, betting, or esports wagering.
  • It does not sell or expose player data publicly.

Data Use Summary

  • Riot account identity after opt-in: PUUID, gameName, tagLine.
  • League of Legends custom match data after opt-in: match identifier, champion, KDA, damage, gold, vision score, CS, items, and summoner spells.
  • Local getData usage: private match registration, migration candidates, MVP/ACE records, rankings, performance scores, and monthly/session summaries.
  • Telegram usage: private display of generated summaries to authorized bot users and configured private group chats.
  • Users can disconnect their Riot account and request deletion of stored mappings or tokens.
This demo package uses only sample data. No real Riot API keys, RSO client secrets, access tokens, or private match records are included.
